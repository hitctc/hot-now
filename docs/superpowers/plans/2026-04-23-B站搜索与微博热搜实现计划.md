# B站搜索与微博热搜实现计划

## 1. 执行约束

- 本计划只覆盖第 `4` 阶段的两个来源：`B 站搜索` 与 `微博热搜榜匹配`
- 本计划虽然统一规划两个来源，但实际实现顺序必须固定为：`先 B 站，后微博`
- 本轮不做 `自动调度`、`内容页二级筛选`、`B 站 UP 主/专栏搜索`、`微博全文搜索`、`其他新增来源`
- `/settings/sources` 继续保持“分区式工作台”，不要把 `B 站`、`微博` 重新揉进普通 `content_sources` 配置表

## 2. 目标

本轮要把第 `4` 阶段拆成两条语义清晰、边界稳定的采集链路：

1. `B 站搜索`
   - 在 `/settings/sources` 中维护独立 query 列表
   - 只支持手动采集
   - 只搜索 `视频`
   - 结果去重后写入现有 `content_items`
   - 结果允许进入 `AI 新讯` 和 `AI 热点`

2. `微博热搜榜匹配（固定 AI 关键词，只进入 AI 热点，不做微博全文搜索）`
   - 不维护后台 query CRUD
   - 只拉取微博热搜榜
   - 用内置固定 AI 关键词集合做匹配
   - 命中内容写入现有 `content_items`
   - 结果只允许进入 `AI 热点`

## 3. 总体方案

### 3.1 B站搜索

- 独立数据表保存 query 配置，例如 `bilibili_queries`
- 运行时读取已启用 query，按优先级和全局固定限制执行搜索
- 搜索结果统一映射到隐藏聚合 source：`bilibili_search`
- 每条视频内容统一写入 `content_items`
- `metadata_json` 保留视频作者、播放量、互动量、命中 query、视频 id、collector 类型等附加信息
- `/settings/sources` 新增独立 `B 站搜索` 分区，支持新增 / 编辑 / 删除 / 启停 / 手动采集 / 查看最近状态

### 3.2 微博热搜榜匹配

- 不新增“关键词配置表”，固定关键词集合直接保存在代码常量中
- 运行时拉取微博热搜榜，再对榜单标题做固定 AI 关键词匹配
- 命中结果统一映射到隐藏聚合 source：`weibo_trending`
- 命中内容统一写入 `content_items`
- `metadata_json` 保留热搜位次、热度值、命中关键词、collector 类型等附加信息
- `/settings/sources` 新增独立 `微博热搜榜匹配（固定 AI 关键词，只进入 AI 热点，不做微博全文搜索）` 分区，只提供手动采集与最近状态展示，不提供关键词编辑

## 4. 技术与落点

- 后端继续复用现有 `Node.js + TypeScript + Fastify + SQLite`
- 前端继续复用现有 `/settings/sources` 单页工作台和 `Ant Design Vue`
- 内容入库继续复用现有 `content_items` 和隐藏聚合 source 策略
- 手动采集入口继续使用独立 action route，不并回默认 `POST /actions/collect`

## 6. 分阶段任务

### 任务 1：补齐 B站 query 数据模型与 repository

目标：
- 新增 `bilibili_queries` 表及迁移
- 明确字段、状态语义和最近执行状态回写

建议字段：
- `id`
- `query`
- `is_enabled`
- `priority`
- `note`
- `last_fetched_at`
- `last_success_at`
- `last_result`
- `created_at`
- `updated_at`

要点：
- 复用 Twitter / HN query 表的建模风格
- 保持最小字段集合，不提前加入单 query 时间窗口或限额
- repository 需要覆盖 CRUD、启停、排序、状态回写和读模型

### 任务 2：实现 B站 API client 与 collector

目标：
- 接好 B 站搜索 client
- 把搜索结果映射为统一候选内容

固定边界：
- 第一版只搜 `视频`
- 第一版只支持后台手动采集
- 第一版使用全局固定限制，例如“每轮最多处理 N 个 query、每个 query 最多取 M 条结果”

映射重点：
- 标题
- 视频 URL
- UP 主
- 发布时间
- 播放/点赞/评论等可用指标
- 命中 query
- 平台唯一 id

要点：
- API client 只负责请求和原始结果规整
- collector 负责 query 级运行、去重、字段映射和错误收口
- 单个 query 失败不能打断整轮 B 站采集

### 任务 3：接通 B站内容入库与独立 action

目标：
- B 站结果入库到现有 `content_items`
- 补上隐藏聚合 source `bilibili_search`
- 暴露独立 action route

要点：
- 必须复用现有 `content_items.source_id` 外键策略
- `metadata_json` 中显式记录 `collector = bilibili_search`
- 增加类似 `POST /actions/bilibili/collect` 的独立手动入口
- 这条 action 不生成日报产物，也不并回默认 `collect`

### 任务 4：补齐 B站 `/settings/sources` 工作台

目标：
- 在系统页加上独立 `B 站搜索` 分区

页面能力：
- 新增 query
- 编辑 query
- 删除 query
- 启停 query
- 手动采集 B站搜索
- 展示最近成功和最近结果

要点：
- 不把 B 站 query 混入普通 source 列表
- UI 结构沿用 Twitter / HN 的独立分区模式
- 交互与状态提示尽量复用现有组件/文案模式

### 任务 5：实现微博热搜榜 API client 与固定关键词匹配

目标：
- 接入微博热搜榜数据源
- 建立固定 AI 关键词匹配逻辑

固定边界：
- 只拉热搜榜
- 不做微博全文搜索
- 不做后台关键词 CRUD
- 只进入 `AI 热点`

要点：
- 固定 AI 关键词集合写成明确常量，并附注释说明用途
- API client 负责榜单获取和规整
- collector 负责关键词匹配、结果过滤和状态回写
- 匹配逻辑要尽量透明，便于后续维护

### 任务 6：接通微博热搜榜入库、独立 action 与系统页分区

目标：
- 微博命中结果入库到 `content_items`
- 维护隐藏聚合 source `weibo_trending`
- 增加独立 action route 和系统页分区

要点：
- `metadata_json` 中显式记录 `collector = weibo_trending`
- 通过内容选择规则确保它只进入 `AI 热点`
- 系统页分区标题必须明确显示：
  - `微博热搜榜匹配（固定 AI 关键词，只进入 AI 热点，不做微博全文搜索）`
- 页面只提供：
  - 手动采集
  - 最近成功
  - 最近结果
  - 固定关键词说明

### 任务 7：测试、文档与最终收口

目标：
- 补齐 B站 / 微博的最相关测试和协作文档

测试范围：
- migration
- repository
- API client
- collector
- run 层
- server action routes
- settings API/service
- `/settings/sources` 页面交互

文档范围：
- `README.md`
- `AGENTS.md`

要点：
- 文档里要同步说明这两个来源的定位差异
- 明确 `B 站 = 搜索型来源`
- 明确 `微博 = 热榜匹配型来源`
- 不把“未来可能做全文搜索”写进当前交付说明

## 7. 风险与边界

### 7.1 当前明确不做

- B 站 `UP 主` 搜索
- B 站 `专栏` 搜索
- 微博全文搜索
- 微博关键词后台配置
- 自动调度
- 内容页 B 站 query 二级筛选
- 内容页 微博关键词二级筛选
- 新的来源融合重构

### 7.2 主要风险

- B 站与微博的数据结构差异较大，不能强行抽成一个统一 collector
- 微博热搜榜可用性与接口稳定性可能不如 RSS / HN，需要把失败状态写清楚
- 如果先把微博做成可配置关键词，会明显抬高 `/settings/sources` 复杂度
- 如果一开始就混做两个来源的 UI 和入库逻辑，容易让执行边界失控

## 8. 推荐执行顺序

1. 先完成 `B 站` 的 migration、repository、collector、入库和独立 action
2. 再补 `B 站` 的 `/settings/sources` 分区和测试
3. 在 B 站链路跑通后，再进入 `微博热搜榜匹配`
4. 最后统一补测试、README、AGENTS 和交付验证

## 9. 交付门禁

本计划进入实现前，至少要满足：

- `spec` 已确认
- 本实现计划已确认
- 执行时严格遵守 `先 B 站，后微博`

- 最相关单测通过
- `/settings/sources` 的 B 站与微博分区可正常使用

- `微博热搜榜匹配` 命中结果只进入 `AI 热点`
- `README.md` 与 `AGENTS.md` 已同步更新
