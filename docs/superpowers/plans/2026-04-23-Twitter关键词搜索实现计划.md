# Twitter 关键词搜索实现计划

> **执行约束：** 本计划只覆盖第一阶段 Twitter 关键词搜索。不要顺手接入自动调度、语言地区参数、单条内容隐藏、LLM 过滤、Hacker News、B 站或微博热搜。

**Goal:** 在 `/settings/sources` 增加独立的 `Twitter 关键词搜索` 分区，让用户可以维护关键词列表、手动触发限流搜索，并把命中的 tweet 去重后写入现有 `content_items`，同时支持关键词级 `采集启用 / 展示启用` 双开关。

**Architecture:** 新增 `twitter_search_keywords` 和 `twitter_search_keyword_matches` 两张表保存关键词配置与 tweet 命中关系；采集时通过 TwitterAPI.io `advanced_search` 按优先级选前 `5` 个启用关键词、每词最多处理前 `10` 条结果；所有 tweet 继续复用 `external_id = twitter:{tweetId}` 的全局去重口径，并维护一个隐藏聚合 `content_sources.kind = twitter_keyword_search` 满足 `content_items.source_id` 外键。内容页读取阶段额外判断“是否至少命中一个 `展示启用` 关键词”，但不得反向隐藏已被 Twitter 账号采集命中的内容。

**Tech Stack:** TypeScript, Fastify, Vue 3, Ant Design Vue, SQLite, Vitest

  - 断言每个关键词最多处理前 `10` 条结果。
  - 断言 `401 / 403 / 429 / 空结果` 会生成明确摘要。

- [ ] **Step 2: 扩展 TwitterAPI.io client**
  - 在现有 `twitterApiClient.ts` 中新增 `searchLatestTweetsByKeyword`。
  - 只封装本轮需要的最小参数：
    - `keyword`
    - 固定时间窗
    - `queryType = Latest`
  - 不开启分页，不扩展多页抓取。
  - 不记录真实 `TWITTER_API_KEY`。

- [ ] **Step 3: 实现 collector**
  - 读取 `is_collect_enabled = true` 的关键词。
  - 按 `priority DESC, id ASC` 选择前 `5` 个。
  - 每个关键词调用一次搜索。
  - 每个关键词最多规整前 `10` 条结果。

    - 已启用关键词数
    - 本次执行关键词数
    - 搜索返回 tweet 数

    - 复用已有内容数
    - 失败关键词数

## 状态与异常边界

- `TWITTER_API_KEY` 缺失：手动采集直接返回不可执行结果，不影响 RSS、公众号和 Twitter 账号采集。
- 单个关键词失败：只更新该关键词 `last_result`，其余关键词继续。
- 搜索成功但无新增：必须明确回显为“成功但没有可入库的新推文”。
- 关键词被停用：后续不再参与搜索。
- 关键词被隐藏：历史命中内容按关键词路径不再显示，但不删除入库数据。
- 删除关键词：只删除配置与命中关系，不删除已经入库的 `content_items`。
- 同一 tweet 命中多个关键词：只要还有一个可见关键词，内容继续显示。
- 同一 tweet 同时命中 Twitter 账号采集：关键词隐藏规则不得把它一并隐藏。

## 风险与边界

- 第一版搜索质量会受关键词粒度影响，噪音主要靠小规模手动试错和现有内容页规则控制。
- 第一版不做高级 query 拼装器，避免把简单关键词管理复杂化。
- 第一版不做自动定时采集，避免 credits 在调试阶段被无感消耗。
- 第一版不做多页翻页，避免单次手动触发跨出 `5 × 10` 的预算边界。
- 第一版不做单条内容级隐藏；这属于后续更细粒度的运营能力，不应混进当前最小闭环。
