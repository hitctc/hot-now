# AI时间线重要发布事件升级 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 `AI 时间线` 从“官方源文章列表”升级为“最近 7 天国内外主流 AI 公司重要发布事件时间线”，只默认展示 `S级 / A级` 事件，并用中文说明每条事件为什么重要。

**Architecture:** 继续复用现有 `ai_timeline_events` 独立事件表、`src/core/aiTimeline/` 采集模块、`/api/ai-timeline` API 和 `/ai-timeline` 页面，不把事件写入 `content_items`。本轮通过新增字段、规则评分器、中文摘要生成器和后台轻量编辑动作，把采集到的官方候选自动筛成可展示的重要事件，同时保留人工隐藏、改等级、改摘要的后手。

**Tech Stack:** TypeScript, Fastify, SQLite, Vue 3, Ant Design Vue, Tailwind CSS, Cheerio, rss-parser, Vitest

---

## 1. 已确认需求

- 默认展示范围：最近 `7 天`。
- 主时间线只展示重要事件：`S级`、`A级`。
- 普通小更新不进入主时间线，但可以入库为 `B/C`，便于后台排查和后续调规则。
- `官方前瞻` 可以进入时间线，但必须显式标注 `尚未正式发布`。
- 国内公司必须纳入；第一版优先覆盖现有稳定官方源中的 `Qwen`、`Kimi`、`DeepSeek`、`MiniMax`、`智谱 AI`、`ByteDance Seed`，并新增可读的 `Xiaomi MiMo Blog` 源。
- 摘要必须使用中文，用人话解释“这件事为什么重要”，不照搬英文标题。
- 流程是 `自动抓候选 -> 规则筛选 -> 自动进入时间线 -> 后台可编辑/隐藏`，不是人工确认后才展示。

## 2. 明确不做

- 不接媒体二手新闻、爆料、无官方链接的传闻。
- 不把所有 changelog 小更新展示在主时间线。
- 不引入 LLM 作为第一版必需依赖；中文摘要先用可解释规则模板生成，后续再考虑可选 LLM 润色。
- 不做复杂审核流、多人权限、批量操作和变更历史审计。
- 不伪造暂时无法确认的官方源 URL。可灵 / Kling 可以先使用快手官方新闻稿和其中指向的 Kling release notes 作为官方证据，若后续要做持续自动采集，再单独确认稳定列表页和解析器。

## 5. 数据模型

新增迁移名：

```ts
const aiTimelineEventImportanceMigrationName = "015_ai_timeline_event_importance";
```

新增字段：

```sql
ALTER TABLE ai_timeline_events ADD COLUMN importance_level TEXT NOT NULL DEFAULT 'B';
ALTER TABLE ai_timeline_events ADD COLUMN release_status TEXT NOT NULL DEFAULT 'released';
ALTER TABLE ai_timeline_events ADD COLUMN importance_summary_zh TEXT;
ALTER TABLE ai_timeline_events ADD COLUMN visibility_status TEXT NOT NULL DEFAULT 'auto_visible';
ALTER TABLE ai_timeline_events ADD COLUMN manual_title TEXT;
ALTER TABLE ai_timeline_events ADD COLUMN manual_summary_zh TEXT;
ALTER TABLE ai_timeline_events ADD COLUMN manual_importance_level TEXT;
ALTER TABLE ai_timeline_events ADD COLUMN detected_entities_json TEXT NOT NULL DEFAULT '[]';
```

新增索引：

```sql
CREATE INDEX IF NOT EXISTS idx_ai_timeline_events_visible_recent
ON ai_timeline_events(visibility_status, importance_level, published_at DESC);
```

字段语义：

- `importance_level`
  - `S`：旗舰模型、重大多模态能力、重要开源模型、产品级重大能力。
  - `A`：重要模型小版本、重要 API / SDK / 开发平台更新、重要套餐价格变化。
  - `B`：有意义但不足以进入主时间线的小更新。
  - `C`：低价值噪音，默认不展示。
- `release_status`
  - `released`：已经正式发布。
  - `official_preview`：官方前瞻、preview、roadmap、即将发布，页面标注 `尚未正式发布`。
- `importance_summary_zh`
  - 规则生成的中文摘要。
- `visibility_status`
  - `auto_visible`：规则允许自动展示。
  - `hidden`：后台人工隐藏。
  - `manual_visible`：后台人工恢复展示。
- `manual_title` / `manual_summary_zh` / `manual_importance_level`
  - 后台人工覆盖项；读取时优先使用 manual 字段。
- `detected_entities_json`
  - 存模型名、产品名、能力关键词，例如 `["ChatGPT Image2", "MiMo-V2.5-Pro", "Kling 3.0"]`。

## 6. 重要性规则

新增 `src/core/aiTimeline/aiTimelineImportance.ts`：

```ts
export const aiTimelineImportanceLevels = ["S", "A", "B", "C"] as const;
export const aiTimelineReleaseStatuses = ["released", "official_preview"] as const;
export const aiTimelineVisibilityStatuses = ["auto_visible", "hidden", "manual_visible"] as const;

export type AiTimelineImportanceLevel = (typeof aiTimelineImportanceLevels)[number];
export type AiTimelineReleaseStatus = (typeof aiTimelineReleaseStatuses)[number];
export type AiTimelineVisibilityStatus = (typeof aiTimelineVisibilityStatuses)[number];

export type ImportantAiTimelineClassification = {
  eventType: AiTimelineEventType;
  importance: number;
  importanceLevel: AiTimelineImportanceLevel;
  releaseStatus: AiTimelineReleaseStatus;
  importanceSummaryZh: string;
  detectedEntities: string[];
  visibilityStatus: AiTimelineVisibilityStatus;
};
```

核心函数：

```ts
export function classifyImportantAiTimelineEvent(input: {
  companyKey: string;
  companyName: string;
  sourceLabel: string;
  defaultEventType: AiTimelineEventType;
  title: string;
  summary?: string | null;
  officialUrl: string;
}): ImportantAiTimelineClassification;
```

规则顺序：

1. 先识别官方前瞻：
   - 命中 `preview`、`roadmap`、`coming soon`、`early access`、`预览`、`即将`、`前瞻`。
   - 若同时命中重大模型或能力词，允许评为 `S/A`，但 `releaseStatus = "official_preview"`。
2. 再识别实体：
   - 模型名：`GPT-5`、`GPT-5.5`、`Gemini`、`Claude`、`DeepSeek`、`Qwen`、`Kimi`、`MiMo`、`Kling`、`Veo`、`Sora`、`Llama`、`Mistral`。
   - 能力词：`image`、`video`、`audio`、`agent`、`coding`、`multimodal`、`open-source`、`weights`、`4K`、`原生画质`、`视频模型`、`图像生成`。
3. 再评等级：
   - `S`：公司主模型 + 新版本 / 发布 / 开源 / 上线；或多模态重大能力；或官方新闻稿级别重大发布。
   - `A`：API / SDK / 工具链重要更新；非旗舰但重要模型更新；套餐 / 价格 / 开放范围重要变化。
   - `B`：普通 release notes、普通 bug fix、局部文档更新。
   - `C`：无法提取明确发布动作，或标题明显是维护类小更新。
4. 最后生成中文摘要。

中文摘要示例：

```ts
// 模型发布
"这是 OpenAI 的一次重要模型发布，说明其核心模型能力继续向前推进。对用户来说，它会影响 ChatGPT、API 或相关工具的能力边界；对开发者来说，它可能带来新的模型选择和产品集成空间。"

// 官方前瞻
"这是 Google AI 的官方前瞻信息，尚未代表能力已经正式上线，但它透露了下一阶段模型或产品能力的方向。因为来自官方渠道，并且涉及核心 AI 能力，所以值得提前关注。"

// 视频模型
"这是快手可灵在 AI 视频生成方向的重要发布，重点在视频画质、时长、镜头控制或多模态生成能力上继续升级。对创作者和视频生产场景来说，这类更新会直接影响 AI 视频从玩具走向生产工具的速度。"
```

  importanceLevel: "S" | "A" | "B" | "C";

  importanceSummaryZh: string;
  visibilityStatus: "auto_visible" | "hidden" | "manual_visible";

后台管理读取使用新接口：

```http
GET /api/ai-timeline/admin-events?page=1&level=B&visibility=hidden
```

后台更新使用一个动作，避免碎片 endpoint 过多：

```http
POST /actions/ai-timeline/events/:id/update
```

请求体：

```ts
{
  visibilityStatus?: "auto_visible" | "hidden" | "manual_visible";
  manualImportanceLevel?: "S" | "A" | "B" | "C" | null;
  releaseStatus?: "released" | "official_preview";
  manualTitle?: string | null;
  manualSummaryZh?: string | null;
}
```

校验规则：

- 未登录返回 `401`。
- 不存在事件返回 `404`。
- 非法枚举返回 `400`。
- `manualTitle` trim 后为空时存 `null`。
- `manualSummaryZh` trim 后为空时存 `null`。

- 标题
- 当前等级
- 发布状态
- 可见状态

最小操作：

- 隐藏：把 `visibilityStatus` 改为 `hidden`。
- 恢复展示：把 `visibilityStatus` 改为 `manual_visible`。
- 改等级：写 `manualImportanceLevel`。
- 改中文摘要：写 `manualSummaryZh`。
- 改发布状态：写 `releaseStatus`。

第一版不做：

- 批量操作。
- 审核流。
- 操作历史。
- 手工新增事件。
