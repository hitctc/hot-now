# HotNow UI Refresh Design

## 背景

当前 unified shell 已经完成了多源内容页、系统页、登录态和深浅主题切换的主链路，但视觉风格仍然明显停留在上一轮的“赛博控制台”语义：

- 深色模式是当前默认母版
- 品牌块、导航和卡片都偏重、偏冷、偏设备面板
- 蓝绿荧光色在多处同时承担结构色和强调色
- 首页、内容页、系统页虽然共用一套壳层，但页面气质区分还不够明确

这套方向在“控制台感”上是成立的，但和当前产品定位已经出现偏差。用户希望保留：

- 左侧导航壳层
- 品牌块与项目名称展示
- 高密度信息浏览
- 现有页面与交互能力

同时把整体风格切换为：

- 浅色为主的母版
- 更像前卫科技杂志与精致产品后台的混合体
- 具有实验感，但不再使用赛博蓝绿控制台语义

本轮不是页面结构重构，也不是功能升级，而是一次统一站点的视觉语言重设。

## 目标

### 核心目标

把 unified shell 从“赛博控制台”重构为 `Editorial Signal Desk`：

- 保留统一壳层、导航结构和页面职责
- 改写颜色、排版、卡片层级和品牌表达
- 让首页、内容页、系统页在同一视觉系统下呈现不同气质
- 以浅色模式为设计母版，再翻译出深色模式

### 用户感知目标

用户在进入站点后，应该直观感受到：

- 这是一个内容驱动的科技编辑台，而不是传统 admin 控制台
- 页面信息密度高，但仍然清晰、利落、可信
- 项目品牌存在感明确，但不再压过内容本身
- 首页更有记忆点，内容页更利于扫读，系统页更利于操作

### 非目标

本轮不做下面这些事：

- 不调整路由结构和页面职责
- 不改内容评分、收藏、点赞、点踩等交互语义
- 不改 source 管理、手动采集、手动发信等系统行为
- 不把服务端 HTML 模板改造成前端框架应用
- 不增加外部字体、外部 UI 组件库或额外网络依赖

## 设计结论

### 主方向

本轮唯一主方向定为：`Editorial Signal Desk`

这是“前卫科技杂志”与“高密度产品后台”的折中方案，强调：

- 实验感
- 高级产品感
- 内容优先
- 浅色纸感分层
- 可操作后台而不是展示型 landing page

### 已确认的用户偏好

- 色调：`米白 / 钴蓝 / 亮橙`
- 壳层：保留左侧栏与品牌块
- 页面母版：浅色为主，深色为翻译版
- 版式气质：大胆后台，而不是杂志封面或情报墙
- 文字气质：高级产品感，而不是强烈工业感或老派 serif 杂志感
- 信息密度：高密度
- 轮廓：小圆角
- 动效：局部强动效
- 表面材质：纸感分层
- 品牌块：报头式品牌块
- 卡片策略：首页偏设计化，内容页和系统页偏高密度务实

### 明确不采用的方向

为避免后续实现漂移，本轮明确不采用：

- 赛博控制台式蓝绿发光主题
- 液态玻璃、厚重模糊和大面积玻璃态卡片
- 紫色主导或霓虹感过强的实验视觉
- 纯展示型杂志首页风格
- 传统 SaaS 模板式后台

## 页面级设计策略

### 统一壳层

统一壳层继续保留当前信息架构，但重写视觉层次：

- 左侧栏继续承担一级导航与品牌识别
- 左侧栏整体视觉减重，不再像深色设备面板
- 主内容区成为视觉主角
- 品牌块保留在左上，但从“控制台品牌块”改为“报头式品牌块”

### 品牌块

品牌块采用“刊头 / 报头”语义，而不是 logo 容器语义：

- 项目名是品牌块主标题
- 仅保留一行短副标题描述定位
- 品牌块重点依靠排版、字号、字距和轻分割线建立识别
- 不依赖厚重背景、霓虹光效或夸张图形

品牌块需要满足两件事：

- 进入页面第一眼能看见项目名
- 不压缩导航和正文的呼吸空间

### 首页 `/`

首页承担整站第一眼记忆点，是最设计化的一页：

- 顶部改为“热点信号区 + 精选内容区”的编辑台结构
- 允许更强的标题层级、重点卡、分区标签和版式节奏
- 首页卡片允许更明显的色条、层叠、精选标识和局部动效
- 仍然保持可操作后台语义，不引入无关大 Banner 或营销式 Hero

### 内容页 `/articles` 与 `/ai`

内容页优先扫读效率和信息密度：

- 保留统一风格，但减少首页的展示欲
- 强调标题、来源、发布时间、评分、摘要、标签的阅读顺序
- 卡片更扁、更紧凑、更稳定
- `/articles` 偏深读与摘要完整性
- `/ai` 偏时效信号与快速浏览

### 系统页 `/settings/*`

系统页沿用同一配色和分层语言，但视觉表达偏“精致产品后台”：

- 弱化编辑化卡片装饰
- 强化表单、状态、按钮、分区标题的可操作性
- 保持小圆角、细边线、纸感层级
- 避免系统页为了统一风格而牺牲效率

## 卡片系统

### 总体策略

采用混合卡片体系，而不是全站只用一套卡片：

- 首页主卡
- 内容标准卡
- 系统操作卡

三类卡片共享同一视觉骨架，但在密度、强调方式和信息节奏上有差异。

### 首页主卡

首页主卡承担精选和视觉记忆点：

- 卡片尺寸更大
- 标题层级更高
- 允许使用局部蓝橙色块、编号、边注、精选条
- 摘要存在感高于其他页面
- 局部动效可以更明显，但只限少数重点模块

### 内容标准卡

内容标准卡用于 `/articles` 和 `/ai`：

- 更高密度
- 更紧凑的标题、元信息和摘要布局
- 固定阅读顺序保持稳定
- 评分、来源、时间、标签位置逻辑一致
- 收藏 / 点赞 / 点踩继续保留，但会重做为更轻、更精致的工具操作区

### 系统操作卡

系统操作卡用于 `/settings/*`：

- 保留统一色彩语义
- 以操作清晰和状态可读为第一优先级
- 比首页卡更规整
- 比内容卡更强调表单和反馈区域

### 统一骨架规则

所有卡片必须满足以下统一规则：

- 标题始终是第一阅读层
- 来源、时间、评分、标签遵循一致的层级顺序
- 小圆角而非硬切角
- 使用细边线、轻阴影和纸感层叠建立深度
- 不使用厚重玻璃模糊、霓虹发光和大面积彩色底板

## 色彩与材质系统

### 色彩比例

整站色彩按照下列关系组织：

- `80%` 纸面中性色
- `15%` 钴蓝结构色
- `5%` 亮橙信号色

### 颜色职责

米白底负责：

- 页面底色
- 卡片承托面
- 纸张与工作台氛围

钴蓝负责：

- 导航激活态
- 一级标题与结构强调
- 分区线和信息边界
- 标签、链接和结构化重点

亮橙负责：

- 关键动作
- 热点提示
- 状态提醒
- 少量高优先级信号点火

### 材质表达

材质方向为“纸感分层”，不是“亮面玻璃”：

- 背景更像样刊纸面与工作台
- 卡片通过轻阴影、细边线和层级差建立深度
- 允许少量分区编号、短色条、边注形成编辑整理感
- 不做大面积渐变炫光

## 字体与排版系统

### 字体策略

本轮采用“报头式标题 + 产品化正文 + 局部技术感数字”的混合策略：

- 品牌块与少量一级标题具备刊头气质
- 页面正文、导航、表单和按钮继续维持产品化 sans 可读性
- 时间、分数、编号和少量标签可带轻度 mono / technical 气质

### 字体约束

由于项目当前不应增加外部网络依赖，本轮不把 Google Fonts 作为默认实现前提。

这意味着设计目标先落在：

- 字重
- 字号比例
- 字距
- 行高
- 中英数混排节奏

如后续明确允许引入本地托管字体资源，可再单独立项扩展。

### 排版原则

- 高密度不等于拥挤
- 标题、摘要、元信息之间必须有明确层级差
- 正文和摘要保持可读行高
- 分区标题和小标签承担“编辑判断”而不是纯装饰

## 动效策略

### 动效边界

本轮动效遵循“局部强动效，全局克制”的原则：

- 首页品牌区与重点卡可有更明确的进入动画
- 卡片 hover 以边线、阴影、标题色和色条变化为主
- 主题切换允许轻微的色温翻转感
- 设置页和高频操作区动效明显收敛

### 动效约束

- 每页只有 `1-2` 个重点元素承担明显动效
- 默认过渡时间控制在 `180-280ms`
- 避免缩放过大、位移过远或滚动驱动动画
- 必须尊重 `prefers-reduced-motion`

## 实现边界

### 主要改动文件

本轮优先改动以下文件：

- `src/server/renderAppLayout.ts`
- `src/server/renderContentPages.ts`
- `src/server/renderSystemPages.ts`
- `src/server/public/site.css`
- `src/server/public/site.js`

### 实现顺序

实现建议顺序如下：

1. 先重建 `site.css` 中的全局 token、浅深主题关系和基础排版
2. 再调整 `renderAppLayout.ts` 的品牌块与导航层级
3. 再改 `renderContentPages.ts`，建立首页主卡与内容标准卡差异
4. 再改 `renderSystemPages.ts`，把系统页收敛成精致产品后台
5. 最后在 `site.js` 或 CSS 中补局部动效与主题切换细节

### 不应出现的实现漂移

实现时需要明确避免：

- 首页被做成 marketing hero
- 内容页退化成普通列表页
- 系统页继续沿用重控制台风格
- 浅色模式只是把深色 token 简单翻转
- 亮橙色大面积滥用

## 验收标准

### 视觉验收

- 第一眼不再是赛博控制台
- 左上品牌块明确、精致、可识别
- 浅色模式成立且是主设计
- 首页明显比内容页更有记忆点
- 系统页明显比首页更务实

### 可读性验收

- 高密度下仍能快速扫描标题、来源、时间、评分和摘要
- 正文与元信息对比度足够
- 蓝橙强调不影响正文阅读
- 主题切换后浅深模式都保持完整语义

### 交互验收

- 收藏 / 点赞 / 点踩语义不变
- source 管理与手动操作语义不变
- hover / focus 可感知但不过度
- `prefers-reduced-motion` 生效

### 回归验收

- `/`
- `/articles`
- `/ai`
- `/settings/view-rules`
- `/settings/sources`
- `/settings/profile`

以上页面都需要在浅色和深色下进行目检或自动化验收。

legacy 页面继续共享主题资源，但不是本轮视觉升级的主设计对象；它们只需要保证不被共享 token 回归打坏。

## 设计来源与取舍说明

本轮在收口阶段参考了 `ui-ux-pro-max` 的检索结果，主要保留了下面的有效结论：

- 内容优先
- 浅色纸感背景
- editorial / premium 的排版方向
- 高密度场景下优先保证对比度、行高和 reduced motion

但明确放弃了它自动推荐的不适配项：

- `Liquid Glass`
- 暖棕色主色方案
- 依赖外部字体的默认实现路径

最终设计以本项目现有约束和用户确认偏好为准，而不是直接采用检索工具的默认模板。
