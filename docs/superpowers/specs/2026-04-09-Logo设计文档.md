# HotNow Logo Design

## 1. 目标

为 `HotNow` 设计一枚可直接用于 app 图标场景的纯图标 logo。

这轮目标很收敛：

- 先做一个简洁、稳定、识别度高的图标
- 优先适配小尺寸显示
- 同时产出高清版和普清版

这轮不做：

- 带字标版本
- 横版 logo
- 多配色品牌系统
- favicon 多尺寸包
- SVG 矢量重绘版

## 2. 已确认方向

### 2.1 品牌气质

logo 走 `简洁` 路线，不做复杂装饰，不做过多细节堆叠。

### 2.2 图形类型

logo 采用 `纯图标` 方案，不带 `HotNow` 字标，优先适配 app 图标、头像裁切和小尺寸入口。

### 2.3 核心语义

核心识别来自 `热点 / 信号波纹`，不走字母变形、闪电或时钟方向。

### 2.4 外轮廓

整体外轮廓采用 `圆角方形`，保证：

- 更适合 app 图标语义
- 小尺寸更稳
- 后续接入网页角标和启动图更自然

## 3. 最终设计方案

本轮采用单一推荐方案：

`中心热点 + 单圈扩散`

结构如下：

- 外层是圆角方形底板
- 中间是一个清晰的热点信号点
- 外围是 1 到 2 圈极简信号波纹
- 整体保持几何、对称、干净

限制条件：

- 不做写实雷达
- 不加复杂纹理
- 不使用过多细碎线条
- 不依赖文字才能识别

目标观感：

- 一眼看出“热点 / 信号”
- 缩小后仍能认出来
- 放在深浅背景上都不会失真

## 4. 配色规范

### 4.1 主配色

- 底板：`深墨蓝 / 近黑`
- 信号元素：`亮青蓝 / 电光蓝`

### 4.2 配色原则

- 高对比
- 少颜色
- 不做彩虹渐变
- 不依赖复杂阴影

这样处理的原因：

- 更符合 HotNow 当前的科技资讯产品气质
- 小尺寸下不会糊
- 高清版和普清版都容易保持一致性

## 5. 输出规格

### 5.1 高清版

用途：

- README 展示
- 文档封面
- 品牌主视觉源文件
- 后续再缩放输出

规格：

- 尺寸：`1024x1024`
- 文件名：`hotnow-logo-hd.png`

设计要求：

- 可保留更完整的波纹结构
- 可保留轻微光感
- 边缘打磨允许更精细

### 5.2 普清版

用途：

- app 图标预览
- 小尺寸入口
- 日常快速接入

规格：

- 尺寸：`256x256`
- 文件名：`hotnow-logo-sd.png`

设计要求：

- 减少细节
- 保证 `64x64`、`128x128` 下仍可识别
- 不依赖极细描边和微弱光效

### 5.3 两版关系

高清版与普清版不是两套不同 logo，而是同一套设计语言的两种输出：

- 高清版更完整
- 普清版更收敛

## 6. 落地产物

本轮最终应产出两个 PNG 文件：

- `hotnow-logo-hd.png`
- `hotnow-logo-sd.png`

默认应写入当前项目可直接引用的目录，而不是只停留在临时生成目录中。

## 7. 验收标准

生成完成后，至少满足下面标准：

- 图标在 `1024x1024` 下边缘干净
- 图标在 `256x256` 下清晰可读
- 缩到小尺寸后，中心热点与波纹关系仍然成立
- 不会让人误解成雷达、闹钟或闪电图标
- 风格简洁，不复杂，不花哨

## 8. 后续不在本轮范围内的扩展

如果这枚 logo 方向成立，后续可再继续扩展：

- 带字标版
- favicon 套装
- SVG 版
- 深色背景版 / 浅色背景版微调
- 登录页或侧边栏中的品牌接入
