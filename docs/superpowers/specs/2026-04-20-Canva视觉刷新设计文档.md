# HotNow Canva-Inspired Visual Refresh Design

## 1. 背景与目标

HotNow 当前统一站点已经完成信息架构和功能闭环，但整体视觉仍然偏 `Notion` 风格的黑白灰工具界面：层次克制、背景平、装饰极少、强调色弱、登录页与内容页缺少“第一眼抓住用户”的舞台感。  

本轮设计目标不是做一次局部美化，而是把全站升级为一套**高度借鉴 Canva 首页气质**、同时仍适合内容产品长期使用的视觉系统。

本轮明确共识如下：

- 覆盖范围：全站，包括 `/`、`/ai-new`、`/ai-hot`、`/settings/*`、legacy `/history`、`/reports/:date`、`/control`，以及 `/login`
- 参考强度：高度借鉴 Canva
- 信息密度策略：允许适当降低信息密度，换更强的首屏视觉表现
- 视觉方向：`C / Spotlight Product`
- 主色调：冷感科技，蓝青 / 紫 / 电光感
- 登录页强度：强品牌首屏

本轮设计结论是采用：

**`C2 / 电光聚焦舞台`**

它保留 Canva 的亮点：渐变辉光、产品舞台、漂浮卡片、大圆角、彩色高亮、明显的深浅色氛围差异；但页面骨架仍然服务 HotNow 的内容阅读与系统操作，不做成纯 marketing landing page。

## 2. 设计原则

### 2.1 视觉原则

- 页面应该首先传达“这是一个有品牌完成度的 AI 内容产品”，而不是“一个能用的后台工具”
- 所有主页面都必须存在明确的视觉锚点：hero、聚焦卡、装饰光斑、发光按钮或悬浮信息块
- 卡片、面板、弹窗、输入控件不再使用朴素平面盒子，而要统一升级为更大圆角、更精细边框和更有层次的阴影系统
- 深色模式不是浅色模式的反相；深色应成为主舞台，浅色是同一品牌体系下的高亮版本
- 装饰必须服务结构：允许丰富，但不能把内容区做成杂乱拼贴

### 2.2 产品原则

- 内容页仍以“标题、摘要、来源、反馈”作为核心，不因为视觉升级牺牲阅读效率
- 系统页仍需保持操作效率，不能把表单页做成纯展示页
- legacy 页面必须在同一视觉体系中降级重绘，而不是继续保留旧时代样式
- 登录页允许最强视觉表现，但登录动作本身必须始终清晰、稳定、可聚焦

### 2.3 工程原则

- 尽量沿用现有 `editorialTokens.ts + tailwind.css + Ant Design Vue override` 的主题收口方式
- 不引入新的大体量 CSS 皮肤文件；视觉升级主要通过 token、共享布局样式、少量页面级辅助类完成
- 新样式需要同时覆盖深色与浅色，不允许先做单主题、后补另一套
- 优先重做共享壳层、共享 token 和内容页公共组件，再逐页收口系统页与 legacy 页

## 3. 设计方向总述：C2 电光聚焦舞台

### 3.1 整体气质

整体视觉以“科技产品舞台”而不是“创意拼贴墙”为主。页面将采用更强的背景层、冷色高亮、玻璃面板、发光按钮和更大的圆角系统，让用户感受到：

- 产品化
- 高级感
- 未来感
- 视觉张力

但版式仍然保持清晰的阅读顺序，不做过度实验性排版。

### 3.2 核心视觉元素

- 冷色光斑：蓝青、紫、电光蓝的多层 radial gradient
- 舞台背景：页面顶层或 hero 区存在半透明辉光和柔和噪点感
- 聚光卡片：重要卡片采用更强边框、高亮描边、柔和投影和轻微内高光
- 漂浮辅助块：用于展示筛选、统计、状态或 CTA 的小型悬浮信息块
- 发光按钮：主 CTA 在 hover / active / focus 时需要有更明显的高亮扩散感
- 高完成度深浅色：深色更深、更沉、更有舞台感；浅色更亮、更轻，但保留彩色雾感

## 4. Token 与主题系统重构

### 4.1 字体策略

现状 `fontUi` 偏系统工具风。改版后：

- UI 字体仍保持跨平台稳定优先，不额外引入在线字体依赖
- 但字号、字重、字距和标题比例需要重做，形成更接近 Canva 的产品站层次
- 大标题使用更激进的负字距和更高字重
- 小标签与辅助文案使用更统一的 uppercase / tracking 体系

### 4.2 圆角系统

现状圆角偏小，整体保守。改版后：

- `radiusSm`：用于 pill、微型按钮、tag
- `radiusMd`：用于输入框、小面板
- `radiusLg`：用于常规卡片和表格容器
- `radiusXl`：用于 hero 卡、主面板、登录框、drawer
- 顶层卡片和 hero 需要明显更圆，视觉上接近 Canva 的软性卡片语言

### 4.3 色彩系统

新增或强化以下 token 族：

- `accentPrimary`: 电光蓝
- `accentSecondary`: 冷紫
- `accentCyan`: 青蓝高亮
- `accentGlowA/B/C`: 用于背景光斑和 hover halo
- `surfaceGlass`: 半透明玻璃面板底色
- `surfaceHero`: hero 主面板底色
- `surfaceElevated`: 悬浮小卡底色
- `borderGlow`: 用于强交互态边框
- `shadowGlow`: 彩色阴影

保留现有语义色命名方式，但不再让 `accent/success/warning/danger` 全都退回黑白灰。

### 4.4 阴影系统

现状阴影很弱。改版后需要三层：

- `shadowCard`: 普通内容卡
- `shadowFloating`: 浮层、弹窗、drawer、导航悬浮块
- `shadowGlow`: hero CTA、重点卡片、交互 hover 的彩色扩散阴影

阴影需要同时存在：

- 结构阴影：定义层级
- 品牌阴影：传达冷色光感

### 4.5 背景系统

页面背景不再是单一纯色：

- `bgPage` 负责基础底色
- `bgPageGlowA/B/C` 负责主背景中的多层 radial glow
- 可增加弱噪点或超轻微网格纹理，提升质感
- 浅色模式也要保留淡彩色背景雾层，避免回到普通白底

## 5. 页面级视觉方案

## 5.1 内容页：`/`、`/ai-new`、`/ai-hot`

### 目标

让内容页从“信息列表页”升级为“内容产品首页”。

### 方案

- 顶部增加强 hero 区：标题、当前视图说明、精选摘要、关键统计或当前排序/筛选状态
- hero 背后叠加冷色辉光和品牌装饰层
- `ContentToolbarCard` 升级为更有存在感的控制面板，而不是普通工具条
- 内容卡从单调列表升级为更有舞台层次的卡片：
  - 重要卡片更大、更亮、更有高光描边
  - 标准卡仍保持阅读效率，但要有更明显 hover 漂浮感
  - 来源、时间、评分、反馈入口用统一视觉语言收口
- 页面顶部可加入一个品牌感更强的浮动信息条或 spotlight badge

### 约束

- 列表阅读效率不能明显下降
- 搜索、排序、来源筛选仍然要清晰可见
- hero 不应压掉首屏内容太多，需要在大屏和移动端分别优化高度

## 5.2 系统页：`/settings/*`

### 目标

让系统页从“后台工作台”升级为“高完成度产品控制中心”。

### 方案

- 左侧壳层、页头、介绍区统一升级为产品化舞台
- 每个系统页顶部都有独立 hero intro block，包含标题、解释、状态摘要
- 表单区、表格区、动作卡区用不同层级的 surface 表达，避免一片平
- 按钮、二级按钮、危险按钮、tag、状态标签全部统一重绘
- 表格容器采用更大圆角、内阴影、行 hover 提升和更精细的表头处理
- 删除/清空等危险操作在视觉上明确分离，并用危险色与确认流程共同表达

### 页面细化

- `ViewRulesPage`：重点做“策略控制中心”感，分区更清楚，厂商卡与规则卡做更强层级
- `SourcesPage`：强调 source inventory + analytics workbench 的产品感，图表区和库存区风格拉开
- `ProfilePage`：做成小型品牌资料页，而不是简单信息卡

## 5.3 Legacy 页面：`/history`、`/reports/:date`、`/control`

### 目标

即便仍然保留 legacy 功能，也必须进入同一品牌视觉体系。

### 方案

- 统一套用新的页面背景、标题、卡片和按钮样式
- 弱化“旧页面”感，至少在视觉层让它们像同一产品里的次级模块
- 报告页可提升为更强的阅读展示页，增加封面式头部和更清晰的内容节奏

### 约束

- 不要求大改结构，但不能继续保留明显过时的按钮、间距和容器风格

## 5.4 登录页：`/login`

### 目标

做成全站最强视觉入口，体现品牌气质与产品完成度。

### 方案

- 登录页采用双层结构：
  - 一侧为品牌舞台区：大标题、价值主张、冷色背景光斑、漂浮演示卡
  - 一侧为登录表单区：玻璃感或高亮面板，重点突出输入与主 CTA
- 背景可以有更明显的渐变、柔光、抽象卡片和视觉粒子感
- 按钮、输入框、表单焦点态要明显比当前精致
- 浅色模式也不能变成普通白底登录页，必须保留品牌背景层

### 约束

- 登录动作优先级必须始终高于装饰
- 移动端必须压缩为单栏，但保留品牌主视觉的识别性

## 6. 共享骨架与组件升级

## 6.1 Unified Shell

- 侧边栏从当前简洁工具栏升级为更有产品感的 navigation rail
- logo 区、主题切换、底部账号块要做成统一品牌组件
- 移动端顶部导航和系统抽屉也需要继承新的高亮、圆角和阴影系统

## 6.2 共享组件

需要逐一升级的共享组件包括但不限于：

- `UnifiedShellLayout`
- `EditorialPageIntro`
- `ContentToolbarCard`
- `ContentHeroCard`
- `ContentStandardCard`
- `ContentFeedbackPanel`
- `ContentSourceFilterBar`
- `ContentSortControl`
- `ContentSearchControl`
- `EditorialEmptyState`

### 共享规则

- 组件不各自定义视觉语言，统一从 token 和少量共享 class 消费
- hover / focus / active 动画轻量但明确
- 交互反馈需要更“亮”，但不能俗气或过度弹跳

## 7. 深色与浅色模式策略

## 7.1 深色模式

深色模式是主场景：

- 更深背景
- 更明显的蓝紫辉光
- 更强的浮层与玻璃感
- 文字对比强但不刺眼
- 重点按钮和高亮信息拥有明确光感

## 7.2 浅色模式

浅色模式不是普通文档白：

- 使用冷白、雾白、极浅蓝灰作为主底
- 保留淡彩渐变和局部 glow
- 面板之间用层次和轻阴影而不是单纯描边区分
- CTA 与高亮块仍维持电光蓝 / 冷紫气质

## 8. 响应式策略

- 大屏：完整展示 hero、悬浮点缀和双列/多层卡片
- 中屏：hero 缩短、浮层减少、控制区更紧凑
- 小屏：优先保留品牌色和氛围，不强行保留所有装饰块
- 移动端登录页需改为单栏，但仍保留品牌头图 / 背景层 / CTA 高亮

响应式优先级：

1. 先保留层次
2. 再保留品牌感
3. 最后裁剪装饰

不要为了移动端简单而把整体设计退回普通后台样式。

## 9. 实现边界与阶段拆分建议

本轮设计规模较大，不建议一次性无计划铺开。推荐实现阶段：

### Phase 1：主题基建

- 重构 `editorialTokens.ts`
- 更新全局背景与主题变量
- 更新基础按钮、卡片、输入、tag、alert 样式

### Phase 2：统一壳层与登录页

- 重做 `UnifiedShellLayout`
- 重做 `/login`
- 建立全站视觉骨架

### Phase 3：内容页

- 重做内容页 hero、工具条、内容卡和空态
- 收口深浅色下的主要视觉语言

### Phase 4：系统页与 legacy 页

- 升级 `/settings/*`
- 升级 `/history`、`/reports/:date`、`/control`

## 10. 测试与验收标准

## 10.1 设计验收

- 首页、登录页、系统页首屏都有明确视觉锚点
- 深浅色都能明显感受到品牌统一性，而不是一套强一套弱
- 用户第一眼能够感受到“这是 Canva 气质的产品化页面”，但仍然认得出是 HotNow

## 10.2 前端验收

- 内容可读性没有明显下降
- 关键功能入口没有被装饰遮挡
- 深浅色切换后关键组件状态正确
- 移动端布局不崩
- legacy 页面视觉不再脱节

## 10.3 验证建议

- 最相关前端单测
- `npm run build:client`
- 全站手动 smoke test：
  - `/login`
  - `/`
  - `/ai-new`
  - `/ai-hot`
  - `/settings/view-rules`
  - `/settings/sources`
  - `/settings/profile`
  - `/history`
  - `/reports/:date`
  - `/control`
  - 深浅色切换
  - 移动端导航和系统抽屉

## 11. 非目标

本轮不包含：

- 改变现有业务流程和路由结构
- 新增功能模块
- 接口改造
- 内容页信息架构重排
- 大规模前端技术栈变更

本轮核心任务是：

**把现有 HotNow 全站视觉从朴素工具风，升级为高度借鉴 Canva、但仍适合内容产品的“电光聚焦舞台”体系。**
