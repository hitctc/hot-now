# 2026-05-08 内容创作流程设计文档

## 1. 背景与目标

hot-now 当前只做「内容采集 + 评分 + 展示」，不具备将采集到的新闻素材进一步加工为成品文章的能力。

本阶段的目标是在 hot-now 中新增「内容创作」模块，实现从「素材收集」到「成品文章展示与多格式复制」的完整链路。hot-now 本身不调用 LLM 写文章——写作由外部 worker（Codex / ChatGPT 等）完成。hot-now 的职责是：

1. 接收外部 agent 推送的原始新闻素材
2. 展示素材列表和状态
3. 接收外部 worker 推送的成品文章
4. 展示成品文章，支持编辑、状态流转和纯文本一键复制

本阶段明确不做的：

- 不在 hot-now 内部集成 LLM 调用
- 不接各平台发布 API
- 不做平台适配渲染（公众号 HTML / 小红书 / 抖音等），第一版只做纯文本一键复制

## 2. 本阶段结论

### 2.1 架构

新增独立模块「内容创作」，与现有「内容」（AI 新讯 / AI 热点 / AI 时间线）和「系统」（设置页面）并列。需要登录才能访问。

「内容创作」下拆两个子页面：

1. **素材库** — 展示 agent 推送的原始素材，可查看关联的成品文章
2. **成品文章** — 展示 worker 推送的成品文章，支持状态流转和纯文本复制

两个模块通过双向外键关联：素材指向已生成的成品文章，成品文章指向来源素材。

### 2.2 数据流

```
外部 Agent ──POST──▶ /api/creative/source-items ──▶ creative_source_items
                                                            │
外部 Worker 读素材 ────────────────────────────────────────┘ │
       │                                                     │
       ▼                                                     │
外部 Worker ──POST──▶ /api/creative/finished-articles        │
                            │                                │
                            ▼                                │
                   creative_finished_articles ──FK──▶ sourceItemId
                            │
                            ▼
                   前端展示 + 状态流转 + 纯文本复制
```

### 2.3 实现顺序

1. 数据库 migration + repository 层
2. 素材推送 API + 素材库页面
3. 成品文章推送 API + 成品文章页面
4. 关联关系展示（素材 ↔ 成品文章）
5. 纯文本一键复制
6. 状态流转操作

## 3. 不采用的方案

### 3.1 不复用 content_items 表

现有 `content_items` 表是为 RSS pull 模式设计的，有 `source_id` 外键绑定到 `content_sources`，字段结构服务于采集流水线。素材推送是 push 模式，字段需求和生命周期完全不同。复用会带来不必要的约束和耦合。

### 3.2 不在 hot-now 内部调 LLM

写作能力由外部 worker 提供，hot-now 只做存储和展示。保持职责单一。

### 3.3 不做 Writing Jobs 独立表

写作任务的轻量追踪通过素材表上的 `linkedArticleId` 字段实现。素材有成品文章关联时显示「已生成」，没有时显示「未生成」。不需要额外的任务队列表。

## 4. 数据模型设计

本阶段新增两张表，均以 `creative_` 前缀命名，与现有 `content_*` 表隔离。

### 4.1 `creative_source_items` — 素材库

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK AUTOINCREMENT | |
| externalId | TEXT | NOT NULL | 外部唯一 ID |
| collectorAgent | TEXT | NOT NULL | 推送 agent 标识 |
| title | TEXT | NOT NULL | 标题 |
| url | TEXT | NOT NULL | 原文链接 |
| sourceName | TEXT | | 来源名称 |
| summary | TEXT | | 摘要 |
| fullContent | TEXT | | 清洗后全文内容 |
| author | TEXT | | 作者 |
| coverImageUrl | TEXT | | 封面图 URL |
| tags | TEXT | | 标签（JSON array） |
| language | TEXT | DEFAULT 'zh' | 语言标识 |
| wordCount | INTEGER | | 原文字数 |
| contentType | TEXT | | 文章类型 |
| score | REAL | | 原文分数 |
| publishedAt | TEXT | | 原文发布时间 |
| collectorTimestamp | TEXT | | agent 推送时间 |
| qualityStatus | TEXT | NOT NULL DEFAULT 'pending' | pending / accepted / rejected |
| rawPayloadJson | TEXT | NOT NULL | 原始完整 JSON 照收 |
| linkedArticleId | INTEGER | FK → creative_finished_articles(id) | 关联的成品文章，NULL 表示未生成 |
| createdAt | TEXT | NOT NULL DEFAULT CURRENT_TIMESTAMP | |
| updatedAt | TEXT | NOT NULL DEFAULT CURRENT_TIMESTAMP | |

约束：

- `UNIQUE(externalId, collectorAgent)` — 同一 agent 的同一 externalId 去重

说明：

- `rawPayloadJson` 存推送时的完整请求 body，不丢数据
- normalized fields 从 rawPayloadJson 提取，结构化查询用
- `qualityStatus` 由用户在素材库手动标记，不影响 worker 读取
- `linkedArticleId` 在 worker 推送成品文章时自动回写

### 4.2 `creative_finished_articles` — 成品文章

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK AUTOINCREMENT | |
| sourceItemId | INTEGER | FK → creative_source_items(id) | 来源素材 |
| mode | TEXT | | A（观点文）/ B（观察随笔） |
| thesis | TEXT | | 核心立意 |
| contentMarkdown | TEXT | NOT NULL | 成品正文（Markdown） |
| titles | TEXT | | 备选标题（JSON array，最多 5 个） |
| hooks | TEXT | | 开头钩子（JSON array，最多 3 个） |
| quotes | TEXT | | 可摘句（JSON array，最多 5 个） |
| summary100 | TEXT | | 百字摘要 |
| imagesJson | TEXT | | 图片列表（JSON array） |
| status | TEXT | NOT NULL DEFAULT 'generated' | 状态流转见 4.3 |
| rawResponseText | TEXT | | AI 原始返回文本 |
| createdAt | TEXT | NOT NULL DEFAULT CURRENT_TIMESTAMP | |
| updatedAt | TEXT | NOT NULL DEFAULT CURRENT_TIMESTAMP | |

说明：

- `sourceItemId` 必填，成品文章必须有来源素材
- `imagesJson` 结构：`[{url, position, caption, width?, height?}]`，position 表示在正文中的插入位置（Markdown 行号或段落索引）
- `rawResponseText` 保留 AI 完整返回，用于回溯和调试

### 4.3 成品文章状态流转

```
generated → edited → approved → published → completed
    ↓          ↓          ↓
    └──────────┴──────────┴──→ rejected → edited（重新编辑）
```

| 状态 | 含义 |
|------|------|
| generated | worker 刚推送的初始状态 |
| edited | 用户在 hot-now 中编辑过内容 |
| approved | 审核通过 |
| published | 手动标记已发布到某平台 |
| rejected | 审核拒绝 |
| completed | 全流程完成 |

状态流转规则：

- generated → edited：用户在前端编辑了内容
- edited → approved：用户点击「通过」
- approved → published：用户手动标记已发布
- generated / edited / approved → rejected：用户点击「拒绝」
- published → completed：用户确认完成
- rejected 状态可以重新回到 edited（重新编辑）

## 5. API 设计

### 5.1 认证

推送 API 使用独立的 API Token 认证，通过 `X-Creative-Token` header 传递。Token 值存放在环境变量 `CREATIVE_API_TOKEN` 中。

原因：推送 API 的调用方是外部 agent/worker，不是浏览器，不适合用 session cookie。单独的 token 比 session 更简单、更适合机器对机器场景。

查询 API 和操作 API 复用现有的 session 认证机制。

### 5.2 素材推送接口

```
POST /api/creative/source-items
Headers: X-Creative-Token: <token>
Content-Type: application/json

Body（normalized fields，按需传）：
{
  "externalId": string,       // 必填
  "collectorAgent": string,   // 必填
  "title": string,            // 必填
  "url": string,              // 必填
  "sourceName": string,
  "summary": string,
  "fullContent": string,
  "author": string,
  "coverImageUrl": string,
  "tags": string[],
  "language": string,         // 默认 "zh"
  "wordCount": number,
  "contentType": string,
  "score": number,
  "publishedAt": string       // ISO 格式
}

Response:
- 201: { id, externalId, created: true }
- 200: { id, externalId, created: false }  // 已存在，幂等返回
- 400: { error }  // 缺少必填字段
- 401: { error }  // token 无效
```

说明：

- 幂等：`externalId + collectorAgent` 已存在时不报错，返回 200
- normalized fields 从 body 直接提取，同时完整 body 存入 `rawPayloadJson`
- 字段不卡死：必填只有 externalId、collectorAgent、title、url，其余均可选

### 5.3 成品文章推送接口

```
POST /api/creative/finished-articles
Headers: X-Creative-Token: <token>
Content-Type: application/json

Body:
{
  "sourceExternalId": string,   // 必填，用于查找素材
  "collectorAgent": string,     // 必填，与 sourceExternalId 联合定位素材
  "mode": string,               // A / B
  "thesis": string,
  "contentMarkdown": string,    // 必填
  "titles": string[],
  "hooks": string[],
  "quotes": string[],
  "summary100": string,
  "images": [{ url, position, caption, width?, height? }],
  "rawResponseText": string
}

Response:
- 201: { id, sourceItemId, created: true }
- 404: { error }  // 找不到对应素材
- 409: { error }  // 该素材已有成品文章
- 401: { error }
```

说明：

- 推送成功后自动回写 `source_items.linkedArticleId`
- 一个素材只允许关联一篇成品文章（1:1）

### 5.4 查询接口（前端用，需登录）

```
GET /api/creative/source-items?page=1&pageSize=20&qualityStatus=&collectorAgent=&search=
GET /api/creative/source-items/:id

GET /api/creative/finished-articles?page=1&pageSize=20&status=&search=
GET /api/creative/finished-articles/:id
```

### 5.5 操作接口（需登录）

```
POST /actions/creative/source-items/:id/quality-status
Body: { qualityStatus: "accepted" | "rejected" }

POST /actions/creative/finished-articles/:id/status
Body: { status: "edited" | "approved" | "published" | "rejected" | "completed" }

PUT /actions/creative/finished-articles/:id
Body: { contentMarkdown?, thesis?, titles?, hooks?, quotes?, summary100?, images? }
```

## 6. 前端设计

### 6.1 路由与菜单

路由新增：

- `/creative/source-items` — 素材库
- `/creative/finished-articles` — 成品文章

菜单新增独立分区「内容创作」，与「内容」（content）和「系统」（system）并列。需要登录才可见。

导航结构：

```
内容
  AI 新讯
  AI 热点
  AI 时间线
内容创作          ← 新分区
  素材库
  成品文章
系统
  数据收集
  AI 时间线 feed
  筛选策略
  当前用户
```

### 6.2 素材库页面

布局：

- 顶部筛选栏：质量状态筛选、来源 agent 筛选、关键词搜索
- 主体：表格列表，支持分页
- 点击行展开详情

表格列：

- 标题
- 来源（sourceName）
- 评分（score）
- 来源 agent（collectorAgent）
- 发布时间（publishedAt）
- 状态（qualityStatus 徽章）
- 成品状态（linkedArticleId 是否为空 → 「未生成」/「已生成」可点击跳转）

详情展开区：

- 摘要
- 全文内容（可折叠）
- 原文链接（外链）
- 作者、字数、语言、标签
- 关联成品文章卡片（如果已生成，显示标题、状态、创建时间，点击跳转）
- rawPayloadJson（调试用，可折叠）

### 6.3 成品文章页面

布局：

- 顶部筛选栏：状态筛选、关键词搜索
- 主体：卡片视图或列表视图，支持分页
- 点击卡片进入详情

卡片展示：

- 标题（titles[0]）
- 来源素材标题
- 模式（A/B）
- 状态徽章
- 创建时间
- 操作按钮（状态流转）

详情页：

- 来源素材信息卡片（标题、来源、摘要、原文链接）
- 备选标题列表
- 核心立意
- 正文（Markdown 渲染）
- 开头钩子
- 可摘句
- 百字摘要
- 图片列表（缩略图 + URL）
- 纯文本一键复制按钮
- 编辑按钮（可编辑 contentMarkdown、thesis、titles、hooks、quotes、summary100）
- 状态操作按钮

### 6.4 纯文本一键复制

第一版实现：

- 成品文章详情页提供「复制纯文本」按钮
- 将 contentMarkdown 转为纯文本（去掉 Markdown 标记），保留段落结构
- 复制到剪贴板，给出成功提示
- 不做平台适配，不改变内容结构

## 8. 数据库 migration

schema 版本从 17 升级到 18，新增两张表。

Migration 内容：

1. 创建 `creative_source_items` 表
2. 创建 `creative_finished_articles` 表
3. 为 `creative_source_items` 创建索引：`qualityStatus`、`collectorAgent`、`createdAt`
4. 为 `creative_finished_articles` 创建索引：`status`、`sourceItemId`、`createdAt`

## 9. 测试与验收

### 9.1 后端测试

至少覆盖：

- migration 正确执行（两张表、索引、约束）
- 素材推送：正常插入、幂等去重、缺少必填字段拒绝、token 认证
- 成品文章推送：正常插入、回写 linkedArticleId、素材不存在拒绝、素材已有成品拒绝
- 查询接口：分页、筛选、搜索
- 操作接口：状态流转规则校验、编辑更新

### 9.2 前端测试

至少覆盖：

- 路由注册和菜单显示（登录/未登录）
- 素材库列表渲染、筛选、分页
- 成品文章列表渲染、状态筛选
- 详情页渲染（Markdown 内容、关联素材）
- 状态操作交互

### 9.3 验收标准

1. 外部 agent 可以通过 API 推送素材，幂等不报错
2. 素材库页面可查看、筛选、搜索素材
3. 素材可标记质量状态（accepted / rejected）
4. 外部 worker 可以通过 API 推送成品文章，自动关联素材
5. 素材库可看到关联的成品文章状态和链接
6. 成品文章页面可查看、编辑、流转状态
7. 纯文本一键复制可用
8. 未登录用户看不到「内容创作」菜单和页面

## 10. 本阶段明确不做

- LLM 调用（写作在 hot-now 外部完成）
- 平台适配渲染（公众号 HTML / 小红书 / 抖音等）
- 发布 API 对接
- 图片缓存 / 代理（直接使用外部 URL）
- 自动调度（采集和写作都由外部触发）
- 素材自动去重合并（不同 agent 推同一篇文章视为不同条目）
- 成品文章版本管理（只保留最新编辑内容）
